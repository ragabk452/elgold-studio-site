# 🏛️ ELGOLD STUDIO V2 — الخطة الرئيسية (Master Plan)

> دمج الأجزاء الخمسة اللي أرسلتها (Vision · Design System · UI Layout · Reference Guide · Home Blueprint) + مراجعة دقيقة + خطة تنفيذ.
> النص الحرفي الكامل محفوظ في [brief.md](brief.md). الملف ده هو المرجع الرئيسي للشغل.

## 1) الهوية والهدف
- **البراند:** ELGOLD STUDIO — يحس إنه **استوديو تكنولوجيا/سوفتوير premium**، مش بورتفوليو شخصي.
- **الهدف الأوحد:** جذب **عملاء عالي الجودة** (Lead generation). كل قسم بيزوّد الثقة، كل سكرول بيزوّد الفضول.
- **المعيار:** أول 3 ثواني = WOW. الزائر يحس إن الموقع اتكلّف آلاف الدولارات.
- **القاعدة الذهبية:** الإلهام من المرجع (مود/حركة/مسافات/تفاعل)، **مش نسخ** — تجربة ELGOLD أصلية.

## 3) نظام التصميم (Design Tokens) — مرجع سريع
- **Breakpoints/Layout:** Max 1440 · Content 1280 · Reading 720. Section padding 160/120/80. Container 32/24/20.
- **Grid:** 12 / 8 / 4. CSS Grid.
- **Spacing scale (بس دول):** 4,8,12,16,20,24,32,40,48,64,80,96,120,160.
- **Radius:** 10 / 18 / 28 / 36 (hero) / 999.
- **Typography:** Headings 800، line-height 1.05، letter-spacing ضيق بسيط · Paragraph 400، lh 1.8، max 680 · Labels uppercase + wide tracking.
- **Animation speeds:** Fast .25 · Normal .45 · Slow .8 · Premium 1.2. مفيش حركة فورية.
- **Motion:** fade/blur/scale/translate/parallax/reveal — من غير bounce/shake/flash.
- **CSS Variables (إلزامي، مفيش لون hardcoded):** background, surface, card, glass, border, primary, secondary, accent, text, muted, success, warning, danger, glow, gradient-start, gradient-end.
- **Background حيّ:** gradient mesh + grid + noise + glow + particles + moving blur + aurora (حركة بطيئة).
- **Cursor (ديسكتوب):** spotlight + glow + magnetic + scale + cursor-text.
- **Cards:** glass + blur + soft border + glow + radius كبير + tilt خفيف.

## 4) التقنيات
HTML5 + CSS3 + JavaScript (vanilla) + **GSAP + ScrollTrigger** (حركات) + **Lenis** (سكرول ناعم). من غير framework تقيل.
- أداء 60fps: transforms على GPU بس، تجنّب layout thrashing، lazy-load للصور.
- احترام `prefers-reduced-motion` + تبسيط التأثيرات على الموبايل.

## 6) طريقة التنفيذ (قسم قسم + تأكيد)
نبني **قسم واحد**، أصوّره، تأكّد، بعدين اللي بعده. الترتيب:
1. الأساس: tokens.css + base.css + الخلفية الحيّة + اللودر.
2. النافبار.
3. الهيرو.
4. باقي الأقسام بالترتيب (3→12).
5. الموبايل + الأداء + اللمسات الأخيرة.

## 7) ⚠️ المراجعة الدقيقة (نتائج)

### ✅ حاجات متّسقة وسليمة
- الرؤية + نظام التصميم + اللياوت + الإلهام **متناغمين** (دارك premium، glass، glow، CSS vars، Lenis، حركة هادية).
- التقنيات اللي محتاجينها كلها واقعية وتتنفّذ بجودة عالية.
- نظام الـ tokens واضح وكافي للبناء.

### 🟠 تعارضات/غموض بسيط (حليتها مبدئياً)
1. **قائمة الأقسام** ظهرت بـ3 صيغ مختلفة → وحّدتها في القسم (2) فوق.
2. **الهيرو:** البريف بيقول "Center aligned" وفي نفس الوقت "two-column (Left/Right)" → هطبّقه **عمودين داخل حاوية متمركزة** (نص شمال + visual يمين)، وعلى الموبايل عمود واحد.
3. **Desktop-first** (Design System) مع **"Mobile redesigned not resized"** → نبني ديسكتوب-فيرست بس نعمل **معالجة مخصّصة للموبايل** (مش مجرد تصغير).

### 🔴 نواقص مهمة محتاجة قرار منك (قبل/أثناء التنفيذ)
1. **لون البراند (الأهم):** البريف قال "CSS variables بس" لكن **مفيش قيم ألوان محدّدة**. محتاج: لون primary/accent + الجراديانت. (ذهبي يناسب الاسم؟ ولا لون تاني؟)
2. **المحتوى الحقيقي:** الهيكل واضح بس **مفيش محتوى فعلي**:
   - الخدمات (إيه الخدمات بالظبط؟)
   - المشاريع (Featured Projects) — محتاجين **صور أعمال حقيقية**.
   - Testimonials / Client Logos — عملاء حقيقيين ولا placeholders مبدئياً؟
   - أرقام Why ELGOLD (counters) — أرقام حقيقية؟
   - أسئلة FAQ.
   - عناصر Tech Stack.
3. **نظام الطلب/الدفع القديم:** متطلباتك الأصلية (طلب خدمة + تحديد ميزانية بالرقم + تسجيل بالإيميل + دفع PayPal/InstaPay/فيزا + واتساب) **مش موجودة في البريف الجديد**. البريف ده موقع **تسويقي/lead-gen** بينتهي بـ CTA/Contact. **سؤال: نلغي نظام الطلب والدفع، ولا نضيفه (مثلاً في Final CTA / صفحة Contact)؟**
4. **التواصل:** قسم Contact مستقل (فورم؟) ولا نكتفي بـ CTA + Footer؟ وإيه وسيلة التواصل (إيميل/واتساب/فورم)؟
5. **اللغة:** البريف كله إنجليزي. الموقع إنجليزي بس؟ ولا عربي+إنجليزي زي ما اتفقنا قبل كده؟
6. **الخطوط:** نظام التصميم حدّد الأوزان مش اسم الخط. محتاج نختار خط premium (مقترح: Space Grotesk / Geist / Satoshi للعناوين + Inter للنص).

### 🟢 ملاحظات تنفيذية
- التأثيرات كتير (particles + aurora + blur + cursor + tilt) → لازم نراقب الأداء ونطفّي التقيل على الموبايل عشان نحافظ على 60fps.
- Testimonials/Logos لو placeholders، نكتب إنها أمثلة عشان ما توهمش بعملاء مش حقيقيين.

## 8) ✅ القرارات المعتمدة (وصلت من العميل)
1. **الألوان:** Navy blue + Cream. الكحلي = البيئة الداكنة، الكريمي = النص/الأقسام الفاتحة + الأزرار. (CSS variables بالكامل، قابلة للتغيير.)
2. **المحتوى:** مبدئي (placeholders) يتبدّل لاحقاً بأعمال حقيقية.
3. **الدفع:** PayPal + InstaPay (مصر) + فيزا/بطاقة؛ ولو أمكن التحويل يروح PayPal. نظام الطلب/الدفع داخل قسم التواصل/الطلب. (تذكير: قيود PayPal مصر معروفة — UI جاهز والمفاتيح بعدين.)
4. **التواصل:** قسم Contact مستقل (فورم).
5. **اللغة:** إنجليزي (أساسية) + عربي، زرار تبديل + RTL.
6. **الخطوط:** عربي = "فلسطين" (محتاج ملف الخط لاحقاً؛ مبدئيًا Cairo fallback). إنجليزي = Fraunces (عناوين) + Inter (نص).

### 🎨 لوحة الألوان المبدئية (Navy + Cream)
- `--bg` كحلي عميق `#0A1330` · `--surface` `#0E1A44` · `--card` كريمي شفاف
- `--text` كريمي `#F4EEDF` · `--ink` (نص على كريمي) `#0A1330`
- `--primary` كريمي (أزرار على الكحلي) · `--accent` كحلي فاتح/كريمي للتوهّج
- أقسام فاتحة = خلفية كريمي + نص كحلي (للإيقاع heavy/light)
